機能設定：選択中タスクの編集 メニュー

機能設定（タスク）

タスクの名前

ユーザーが識別しやすいように、任意の名前をつけます。実行ファイル名と無関係な名前をつけても、タスクの実行には影響を与えません。
必ず１文字以上の文字列（タスクの名前）を入力すること。（必須入力項目）

実行するファイル名

実際に実行されるファイルの「フルパス名」を入力します。
参照ボタンを押すとファイル選択のダイアログが表示されます。

必ずファイル名を入力すること。（必須入力項目）

実行時のパラメーター

ファイルの実行時に何らかのパラメーター（オプションスイッチ）が必要な場合は、ここに記入してください。

例 ： ノートパッドで、NETLOG.TXTを開く場合は、「実行するファイル名」に「c:\Windows\Notepad.exe」を入力し、「パラメーター」に「c:\Windows\Netlog.txt」と入力します。

作業フォルダ

タスク実行時に作業フォルダを指定します。（実行ファイルのみで有効）
このオプションが設定された場合、「環境設定のカレントディレクトリ」はほとんど意味を持ちません。
※ 適用順序
　　　　環境設定のカレントディレクトリ → 作業フォルダ
通常は実行するプログラムのフォルダを使用

コメント

「タスクを実行する前に、ダイアログを表示する」がONの場合、このコメントが表示されます。
コメントに何も記入されていない場合は、タスクの名前が使用されます。

ウインドウの状態

タスク実行時のウインドウの状態を指定します。
Win32SDK の ShowWindow API のウインドウ状態変数に対応する設定値があります。
通常は「この機能を使わない」

電源の状態

電源状態によってタスクの実行を中止します。
Windows Base Services の Power Management Function の GetSystemPowerStatus より電源状態を取得しています。 通常のノートパソコンなどでの利用を想定しています。 UPSなどを用いたサーバ等での利用は想定していません。
システムの電源状態、OSでの状態取得状況を確認するには「電源テスト」ボタンを押してください。
通常は「常に実行」

実行する

このタスクは実行されます。
このスイッチをOFFにした場合は、実行期日が来ても実行されません。
通常はONで使用

　

機能設定（スケジュール）

ログオン時のスケジュール

「起動ごと」、「日ごと」、「週ごと」、「月ごと」から選択します。
選択された間隔に応じて、すぐ下の間隔を入力します。

例 ： 2週間に1回実行する場合は、「週ごと」と、「２」と入力します。

タスクのスケジュール

特定の曜日に実行しない場合は、その曜日にチェックを入れます。チェックを入れた曜日には（実行期日が来ても）実行しません。タスクの実行期日は順延されます。

また、特定の時間帯に実行したくない場合は、その時間帯を入力してください。2つの時刻に挟まれた時間帯には（実行期日が来ても）実行しません。

　

機能設定（高度な設定）

次のタスクを待機させる

このタスクが実行中は、次のタスクの実行に移りません。このオプションは実行可能ファイル（拡張子がEXE, COM, BAT, CMD のファイル）でのみ有効です。
通常はONで使用

ｘ秒待ってから実行する

このタスクを実行するまで、ｘ秒待ちます。待っている間は、何もしません。
通常は0秒で使用

次の回数実行したら、タスクを停止する

指定された回数実行すると、このタスクの「実行する」スイッチがOFFとなります。OFFになったタスクは設定ダイアログに状態「停止中」が表示されます。
通常はOFFで使用

タスクを実行する前にダイアログを表示する

実行期日が来たとき、タスクの実行前に「タスク ｘｘｘ を実行しますか？」というダイアログを表示します。
通常はOFFで使用

直前のタスクの実行と連動（このタスクの最終実行日時を事前同期）

直前のタスクが実行されたら、このタスクの最終実行日時を現在時にします。
例 ： 全体のバックアップを1ヶ月に1回、重要部分だけのバックアップを1週間に1回する場合、「重要部分だけのバックアップ」は「全体のバックアップ」が行われたときには実行しないというような条件のときに使用するスイッチです。
通常はOFFで使用

直前のタスクと連動

直前のタスクが実行・非実行によりこのタスクを強制実行、実行中止することが出来ます。これは、他の条件判断（経過時間などによる自動実行）よりも優先されます。（例）この連動判断で「実行中止」と判断された場合は、スケジュール上実行するタイミングであっても「実行中止」となります。
通常はOFF（この機能は使わない）で使用

デバッグ用の設定

このタスクの実行履歴を編集できます。おもにデバッグ時に使用することを目的としています。
総実行回数は、実際にタスクが実行された回数。
チェック回数は、タスクの期日をチェックした回数で、実行・非実行にかかわらず毎回カウントアップするカウンターです。

　

　

一時的にOFF

タスクの機能設定の「実行する」をOFFにします。

　

実行

選択中のタスクを実行します。実行期日が来ていなくても実行されます。

　

実行順序「ひとつ上へ、ひとつ下へ」

タスクリストの表示順序（実行もこの順番に行われる）を上下させます。
