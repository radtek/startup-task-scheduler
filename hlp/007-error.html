エラーと対処方法

起動時に起こるエラーは次の通りです

「無効なファイル MFC42.DLL xxxx へのエクスポート」
「…」（メッセージの無いダイアログ）

最新のMFC42.DLLとMSVCRT.DLLをインストールしてください。

「cant create empty document」

ｓTask version 1.5で解決したと思っていますが、まだこのメッセージが出る場合は、このバージョンでは解決できません。（Windows95の場合起こりえます）

「一般保護違反」

sTask version 1.5で解決したと思っていますが、まだこのメッセージが出る場合は、このバージョンでは解決できません。

「Internal Error (Fail to Create Progress Dialog Thread)」

sTask プログラム内部でのエラーです。原因に心当たりがあれば、ぜひ作者までご一報を！ （直すことが出来るかもしれません）
ダイアログ系統のエラーの場合は、タスクの実行自体は正常に行われます。（表示が出ないだけです）

タスク実行時に起こる可能性のあるエラーは、次のとおりです。

「タスク ｘｘｘ は実行できませんでした。ファイルが存在しないか壊れている可能性があります。」

このエラーが表示された場合、タスクの機能設定でファイル名が正しく入力されているか確認してください。

「タスク ｘｘｘ は実行日時に達していないため、実行されません。」

このエラーは、「判定結果ダイアログを表示する」モードでの表示結果です。この場合、スケジュールの条件が整うまでタスクは実行されません。

その他

登録されたタスクのアイコンの色がおかしい

このプログラムは、16色アイコンにしか対応していません。

「ヘルプファイルの表示が出来ません」

ヘルプファイル（sTask.chm）が、sTask.exeと同じフォルダに存在しない場合。もしくは、コンテキストNoが存在しないヘルプ項目を呼び出そうとした場合に表示されるエラーです。
コンテキストIDが0x8000番台 （メニューの各項目） に対する F1 キーでのヘルプは存在しません。（バグでは無く、仕様）

「レジストリに登録（ユーザごと設定）」で、自動実行されない

HKCU\Software\Microsoft\Windows\CurrentVersion に run というキーが存在しない場合に、run の下にある値を書き込もうとして失敗したときに起きるエラーです。
インストール直後のWindowsでは、HKCUのrunには何もかかれていない （runのキーそのものが存在していない）ためにおきます。回避するためには、レジストリエディタで run キーを作成してください。
